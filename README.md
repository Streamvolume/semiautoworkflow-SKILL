## The Problem

科研数据处理中，总有那么一类任务：

它们不够**规整**，以至于你写个 `for` 循环配合 API 就能解决；
又不够**复杂**，以至于值得雇个 RA 或申请一个项目来做自动化。

它们是数据管线中的**灰色地带**——比如从某个没有开放 API 的网页数据库里，把 100 个关键词逐个复制进去搜一遍，找到嵌在页面源码里的 JSON，复制出来，打开某个"JSON 转 Excel"的网页工具下载表格，再按某种神秘命名规范重命名，最后剪切到正确的项目文件夹。

你明知道这些操作 90% 是机械劳动，
但又不得不亲自坐在电脑前，
像个**高度受过训练的生物机器人**一样点击、复制、粘贴、切换窗口。

科研中最昂贵的资源不是 GPU 算力，是**你的注意力**。
而你正在用它来执行 Ctrl+C / Ctrl+V。

**这不合理。**

---

## The Philosophy

SemiAutoWorkflow 的核心理念很简单：

> **你做人该做的事（判断、决策、确认），Agent 做机器人该做的事（解析、转换、搬运、归档）。**
> 
> **而你们之间的协作界面，只需要一个剪贴板和几个热键。**

我们称之为 **"剪贴板管线"（Clipboard Pipeline）**：

```
你的 workflow:
  复制关键词        →  按热键  →  Agent 记住你是谁
  复制原始数据      →  按热键  →  Agent 自动: 提取 → 转换 → 命名 → 归档 → 通知你
  继续下一个...
```

不需要写 Selenium 爬虫，不需要逆向 API，不需要维护容易过期的页面解析逻辑。
你只需要**告诉 Agent 规则**，然后继续用你的肉眼做那些真正需要人类智能的判断。

## Four Stages of Automation

不要追求一步到位。渐进式自动化才是正解：

| 阶段 | 名称 | 人做什么 | Agent 做什么 | 适合时机 |
|:---:|:---|:---|:---|:---:|
| **P0** | 纯手工 | 全部 | 旁观记录 | 第一次做，摸流程 |
| **P1** | 剪贴板 | 搜索、判断、复制 | 读取剪贴板 → 解析 → 转换 → 归档 | **最佳起点** |
| **P2** | 热键伺服 | 搜索、判断、复制 | P1 + 全局热键 + 系统通知 + 零打字 | **推荐长期态** |
| **P3** | 半自动浏览 | 仅确认/纠正 | Agent 操控浏览器填充、点击、提取 | 任务极其高频时 |

**本框架聚焦 P1 → P2 的快速落地。** 通常 30 分钟搭建，立即可用。

## How It Works

### 1. 操作元拆解（Atomic Decomposition）

把你手上的脏活拆解成"原子操作"：

```
阶段        操作元                        执行者    难度    收益
─────────────────────────────────────────────────────────────
输入        读取清单/关键词                人        低      ★★★
检索        打开网页 → 搜索 → 确认存在      人        中      ★★★★
提取        复制嵌在源码里的数据            人        低      ★★
转换        JSON → Excel / 表格规范化      Agent     低      ★★★★★
命名        按规则生成文件名                Agent     低      ★★★★
归档        移动到目标目录 + 记日志         Agent     低      ★★★★
```

### 2. 识别杠杆点

**最大的收益往往来自消灭"无意义的网页中转"。**

如果你现在的流程是：
```
复制 JSON → 打开"JSON 转 Excel"网页 → 粘贴 → 点击转换 → 下载 → 去 Downloads 找文件
```

这完全可以被本地 3 行代码替代：
```python
import pandas as pd
df = pd.read_json(clipboard_json)
df.to_excel("output.xlsx", index=False)
```

Agent 的第一个价值，就是帮你消灭这些"为了格式转换而打开的网页工具"。

### 3. 状态机 + 热键

伺服程序以极简状态机运行：

```
┌─────────┐   [Ctrl+Shift+1]   ┌─────────────┐   [Ctrl+Shift+2]   ┌─────────┐
│  IDLE   │ ─────────────────→ │ WAITING_DATA│ ─────────────────→ │  IDLE   │
│ 空闲    │   (记录剪贴板标识)   │ 等待数据     │   (读取→处理→归档)  │ 空闲    │
└─────────┘                    └─────────────┘                    └─────────┘
     ↑                                                                │
     └──────────────── [Ctrl+Shift+4] 重置 ───────────────────────────┘
```

- **平时**：主线程阻塞休眠，CPU 占用为 0
- **触发时**：热键唤醒 → 执行 → 系统通知 → 回到休眠

## Quick Start

### 安装

```bash
pip install pandas pyperclip pypinyin openpyxl keyboard plyer
```

> Windows 用户若热键无效，请以**管理员身份**运行终端。

### 运行

```bash
# 普通模式（控制台显示状态看板）
python tcmsp_server_example.py

# 静默模式（纯后台，只通过系统通知交互）
python tcmsp_server_example.py --silent
```

### 你的工作流变成这样

| 你的动作 | 世界发生了什么 |
|:---|:---|
| 选中关键词，Ctrl+C，按 **Ctrl+Shift+1** | 系统通知："已记录：当归 → Dang Gui.xlsx" |
| 在数据库确认条目存在，F12 复制 JSON | （无事发生，Agent 在后台休眠） |
| 按 **Ctrl+Shift+2** | 系统通知："✅ 归档完成：当归 \| 15条 \| Dang Gui.xlsx" |
| 继续下一个药材... | 自动回到空闲状态，等待下一轮 |

**全程零打字，零控制台交互，零窗口切换。**

## Key Features

- 🧠 **状态机驱动**：`IDLE` → `WAITING` → `IDLE`，流程清晰不会乱
- ⌨️ **全局热键**：`Ctrl+Shift+1/2/4/Q`，不与浏览器冲突
- 📋 **剪贴板即接口**：人只需要"复制"，不需要写代码
- 🔊 **系统通知 + 音效**：成功/失败即时反馈，不需要盯着控制台
- 🌙 **静默模式**：`--silent` 启动后零控制台刷新，纯后台运行
- 🧹 **事件驱动 UI**：严禁 `while True` 循环刷屏，只在热键触发时刷新
- 📁 **自动防覆盖**：文件名冲突自动追加序号
- 📝 **TSV 日志**：每次操作记录时间、原始名、标准名、文件名、行数
- 🏷️ **别名映射**：内置字典解决异名同药、错别字问题

## Golden Rules

1. **不要追求 100% 全自动。** 如果某个判断点（"这个结果对吗？"）需要人眼确认，那就保留它。Agent 的价值是消灭机械劳动，不是消灭人类判断。

2. **剪贴板是最小可行的人机契约。** 不需要 API、不需要 socket、不需要复杂的 IPC。如果两个程序能互相复制粘贴，它们就能协作。

3. **先跑通单次，再伺服化。** 不要一上来就写热键后台。先写一个"复制 → 运行脚本 → 控制台确认"的版本，验证流程正确后再升级为常驻后台。

4. **本地转换优于网页工具。** 如果你现在的流程里有一个"打开 XX 网页做格式转换"的步骤，这几乎总是可以被 pandas/jsonlite 本地替代，且是 ROI 最高的优化点。

5. **命名规则即文档。** 一个好的自动生成文件名（如 `Dang Gui.xlsx`）比 `download (3).xlsx` 更有价值，因为它在文件系统层面就携带了语义。

## Real-World Impact

> 某位研究者需要将 R 脚本输出的 100+ 味药材清单，逐条到某学术数据库检索活性成分。
> 
> 原流程：每味药约 2 分钟，频繁跨应用切换，极易出错，总耗时 **3~4 小时**。
> 
> 使用 SemiAutoWorkflow 后：人在浏览器内完成搜索判断和复制，Agent 接管所有解析/转换/命名/归档。单味药耗时降至 **30 秒**，且全程不需要切换窗口。总耗时 **< 1 小时**。

## When NOT To Use

- 任务有**开放 API** → 直接用脚本调用 API，不需要剪贴板
- 任务**完全不需要人类判断** → 写个 cron job 或 Snakemake 规则全自动跑
- 任务**只做一次** → 手动可能更快
- 数据涉及**隐私/合规要求** → 剪贴板可能被其他程序读取，敏感数据请用安全通道
